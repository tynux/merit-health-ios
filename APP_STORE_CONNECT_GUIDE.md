# App Store Connect 配置指南

## 概述

本指南将帮助您在 App Store Connect 中配置功德圆满应用，并为 App Store 提交做好准备。

## 前置条件

在开始之前，请确保您已经：

1. 拥有 Apple 开发者账号（年费 99 美元）
2. 在 App Store Connect 中创建了应用
3. 准备好了所有必需的资料（图标、截图、隐私政策等）
4. 生成了应用签名证书和预配置文件

## 第一步：访问 App Store Connect

1. 打开 [App Store Connect](https://appstoreconnect.apple.com)
2. 使用您的 Apple ID 登录
3. 点击"我的应用"
4. 选择"功德圆满"应用

## 第二步：应用信息配置

### 2.1 应用名称和副标题

在"应用信息"部分：

- **应用名称**：功德圆满
- **副标题**：通过健康修行获得功德圆满
- **Bundle ID**：com.meritgame.ios（必须与 Xcode 中的 Bundle ID 匹配）

### 2.2 应用类别

- **主类别**：健康与健身
- **次类别**：健康应用

### 2.3 内容评级

1. 点击"内容评级"
2. 填写内容评级问卷
3. 确保选择正确的评级（通常为 4+）

### 2.4 隐私政策

1. 在"应用隐私"部分，添加隐私政策网址
2. 网址：https://meritgame.app/privacy
3. 确保隐私政策在网站上可访问

## 第三步：应用描述配置

在"App Store 页面"部分：

### 3.1 简短描述

```
将日常健康活动转化为功德分数，收集奖章，实现功德圆满的境界。
```

### 3.2 完整描述

使用 `APP_STORE_METADATA.md` 中的完整描述内容。

### 3.3 关键词

添加以下关键词（每个最多 30 个字符）：

1. 健康追踪
2. 健身应用
3. 冥想正念
4. 功德修行
5. 健康生活

### 3.4 支持网址

- https://meritgame.app/support

### 3.5 营销网址（可选）

- https://meritgame.app

### 3.6 隐私政策网址

- https://meritgame.app/privacy

## 第四步：应用截图和预览

### 4.1 上传截图

1. 点击"屏幕快照"
2. 为以下设备尺寸上传截图：

| 设备 | 尺寸 | 数量 |
|------|------|------|
| iPhone 6.7" | 1290x2796 | 5-10 |
| iPhone 5.5" | 1242x2208 | 5-10 |
| iPad Pro 12.9" | 2048x2732 | 5-10 |

3. 为每个截图添加简洁的说明文字

### 4.2 上传预览视频（可选）

1. 点击"预览"
2. 上传 15-30 秒的应用演示视频
3. 视频应该展示应用的核心功能

## 第五步：应用图标配置

1. 点击"应用图标"
2. 上传 1024x1024 像素的应用图标
3. 确保图标符合 App Store 的要求

## 第八步：构建版本配置

### 8.1 上传构建版本

1. 在 Xcode 中构建应用：
   ```bash
   xcodebuild -workspace ios/App/App.xcworkspace \
     -scheme App \
     -configuration Release \
     -derivedDataPath build
   ```

2. 使用 Xcode 或 Transporter 上传构建版本

3. 在 App Store Connect 中选择构建版本

### 8.2 验证构建版本

1. 等待构建版本处理完成（通常需要几分钟）
2. 检查构建版本的详细信息
3. 确保所有必需的信息都已完成

## 第九步：TestFlight 测试（可选但推荐）

### 9.1 创建测试组

1. 点击"TestFlight"
2. 创建内部测试组
3. 添加测试人员的 Apple ID

### 9.2 分发构建版本

1. 选择要测试的构建版本
2. 添加到测试组
3. 等待 Apple 的审核（通常需要 24 小时）

### 9.3 收集反馈

1. 邀请测试人员下载应用
2. 收集他们的反馈
3. 修复发现的问题
4. 上传新的构建版本

## 第十步：提交审核

### 10.1 最终检查

在提交前，请确保：

- [ ] 应用名称和副标题已设置
- [ ] 应用描述已完成
- [ ] 关键词已添加
- [ ] 应用类别已选择
- [ ] 内容评级已完成
- [ ] 隐私政策网址已添加

### 10.2 提交应用

1. 点击"提交以供审核"
2. 确认提交信息
3. 点击"提交"

## 第十一步：监控审核进度

### 11.1 检查审核状态

1. 在 App Store Connect 中查看应用状态
2. 通常审核需要 1-3 天
3. 您会收到电子邮件通知审核结果

### 11.2 处理审核反馈

如果 Apple 拒绝了您的应用：

1. 仔细阅读拒绝原因
2. 进行必要的修改
3. 上传新的构建版本
4. 重新提交审核

## 第十二步：发布应用

### 12.1 审核通过后

1. 在 App Store Connect 中查看应用状态
2. 状态应该显示"准备提交"或"等待发布"
3. 点击"发布此版本"按钮

### 12.2 应用上线

1. 应用将在 App Store 上线
2. 您会收到确认电子邮件
3. 应用将在搜索结果中显示

## 常见问题

### Q: 应用被拒绝了怎么办？

A: 仔细阅读拒绝原因，进行必要的修改，然后重新提交。常见的拒绝原因包括隐私问题、崩溃、不完整的功能等。

### Q: 应用多久会被审核？

A: 通常需要 1-3 天。在假期或高峰期可能需要更长时间。

### Q: 我可以在审核期间更新应用吗？

A: 不可以。在应用被审核期间，您不能提交新的构建版本。

### Q: 应用上线后如何更新？

A: 您可以上传新的构建版本，重复审核流程。用户将自动收到更新通知。

### Q: 如何处理应用崩溃？

A: 在 Xcode 中修复崩溃，上传新的构建版本，重新提交审核。
